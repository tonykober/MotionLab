Ragdoll Dynamics — 深度調研報告

工具概述

Ragdoll Dynamics 是由 Imbalance Ltd.（倫敦）開發的即時物理模擬器，專為動畫師設計。它作用於 transforms（位移/旋轉/縮放）而非頂點，因此能直接整合到現有的動畫控制器上，無需額外設定物理系統。最初為 Maya 開發，現已支援 Blender，並正在開發獨立版本（Ragdoll 4.0）。

解決的問題：

  • 角色次級動態（頭髮、衣物、尾巴）的手動 K 幀工作量巨大
  • 傳統物理模擬器（nCloth/nHair）基於頂點，難以與動畫控制器整合
  • 需要「物理感覺」的動畫（跌倒、碰撞、被擊飛）手工製作極耗時
  • 動畫疊加（overlapping motion）需要大量經驗和時間

功能詳細說明

核心概念：Markers

Ragdoll 的核心是「Marker」系統——將物理屬性標記到骨骼/控制器上，讓物理引擎在現有動畫基礎上疊加模擬結果。

主要功能模組

功能說明
Markers將物理屬性賦予骨骼，定義質量、形狀、摩擦力等
Groups將多個 Marker 分組管理（如「手臂」「腿」「尾巴」）
Solver物理求解器，控制 substeps、重力、品質參數
Environment定義碰撞平面、障礙物
Manipulator即時互動操控角色姿態
Live Mode即時預覽物理效果，不需播放時間軸
Locomotion自動行走/移動模擬
Constraints物理約束（鏈接、距離、固定等）
Fields力場（風、重力調整等）
Record Simulation將模擬結果烘焙為關鍵幀
Snap to Simulation將控制器吸附到模擬位置
Retargeting模擬結果重定向到不同骨架
Level of Detail距離/性能相關的簡化
Cache快取模擬結果
Import Physics匯入/分享物理設定
Cloth Emulation模擬布料效果（用 transform 模擬而非真正布料）

安裝/使用流程

Blender 版安裝

  1. 下載 Ragdoll for Blender（RagdollBlender.zip）：https://learn.ragdolldynamics.com/blender/download/
  2. 下載 Ragdoll Core（物理引擎核心）
  3. Blender → Edit → Preferences → Add-ons → Install → 選擇 zip
  4. 搜尋 "Ragdoll" 並啟用
  5. 安裝 Ragdoll Core 到指定位置
  6. 首次啟動自動開始 30 天試用

Maya 版安裝

  • 以 Maya Module 形式安裝，支援 Windows/Linux/MacOS
  • 下載：https://learn.ragdolldynamics.com/maya/download

基本使用流程


1. 選擇骨骼/控制器 → Assign Marker（賦予物理屬性）
2. 設定 Group（質量、碰撞形狀）
3. 調整 Solver 參數（substeps、品質）
4. 播放時間軸 → 即時預覽物理效果
5. 滿意後 → Record Simulation（烘焙為關鍵幀）
6. 微調結果 → 匯出動畫

優缺點分析

優點

  • ✅ 即時物理：播放即見效果，調參數即時回饋
  • ✅ 直接作用於控制器：不需要額外的物理骨架或代理系統
  • ✅ Record to Keyframes：模擬結果可烘焙為標準動畫曲線，完全可編輯
  • ✅ 非破壞性：在現有動畫上疊加物理，不影響原始 K 幀
  • ✅ SDK 可序列化：物理場景可匯出，用於其他應用程式
  • ✅ Ragdoll 4.0 願景宏大：獨立於 DCC 的架構，支援即時操偶、硬體整合
  • ✅ 教學豐富：文件完善，社群活躍（Discord + Forums）
  • ✅ 影視級品質：已用於《In The Lost Lands》等影片製作

缺點

  • ❌ Blender 版效能較差：官方說是 Maya 版的 1/10 效能（早期版本）
  • ❌ Blender 版功能不完整：約 80% 的 Maya 功能
  • ❌ 非免費：商業使用需付費
  • ❌ 不支援離線啟動：需要網路驗證
  • ❌ 學習曲線：物理參數調整需要經驗
  • ❌ 不直接匯出到引擎：模擬結果需先烘焙為關鍵幀，再透過其他工具匯出

七、Pipeline 整合分析

項目內容
輸入來源Blender/Maya 骨架動畫（已有的基礎動畫 Action）
輸出格式Blender/Maya 骨架動畫（疊加物理後的 Keyframes，Record Simulation）
上游工具Kimodo BVH、ARP Remap 輸出、任何基礎動畫
下游工具ARP Game Export → 引擎；或 Maya FBX 匯出 → 引擎
推薦搭配Kimodo → BVH 匯入 → Ragdoll 疊加次級動態（頭髮/衣物/碰撞）→ Record → ARP Export。AI 動作的物理後處理層。

對「AI 動作量產工作流」的貢獻定位

定位：AI 動作的物理後處理層 + 次級動態自動化


[AI 動作生成] → [基礎動作] → [Ragdoll 物理疊加] → [高品質最終動作]
  (Kimodo)                     ↑ 次級動態           ↑ 碰撞反應
                               ↑ 物理修正           ↑ Overlapping motion

具體貢獻：

  1. AI 動作品質提升：Kimodo 等工具產出的動作常有「漂浮感」，Ragdoll 可加入重力和碰撞反饋
  2. 次級動態批次處理：頭髮、尾巴、斗篷、裙擺等——AI 無法生成這些，Ragdoll 自動疊加
  3. 碰撞互動：AI 生成的動作不知道環境，Ragdoll 可讓角色正確與地面/物件互動
  4. 即時操偶（4.0）：未來可能取代部分手動動畫和 AI 生成，直接用物理操偶即時表演
  5. 量產效率：設定好物理參數後，套用到同類型的多個動畫上只需 Record Simulation

限制：

  • Ragdoll 不生成動作，它是「後處理」和「增強」工具
  • 大量動作的批次處理仍需腳本化（Python API 支援）
  • Blender 版效能可能成為瓶頸（等待 4.0 改善）
