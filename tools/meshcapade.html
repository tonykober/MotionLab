Meshcapade — SMPL 人體模型商業化平台 ⚠️ 已被 Epic Games 收購

> 調研日期：2026-06-21

> 官網：https://meshcapade.com（平台已於 2026/4/18 關閉）

> 公司：Meshcapade GmbH（德國 Tübingen）

> 狀態：⚠️ 2026 年 2 月被 Epic Games 收購，4 月 18 日平台關閉

> SMPL 授權：https://meshcapade.notion.site/The-SMPL-Model-Licenses

一、工具概述

Meshcapade 是 Max Planck Institute for Intelligent Systems 的衍生公司（2018 年成立），將學術界的 SMPL（Skinned Multi-Person Linear Model）人體模型商業化。提供 AI 動作捕捉、3D 人體建模、動作生成的完整 Toolkit。

核心能力一句話：基於 SMPL 的 AI 無標記動捕 + 3D 人體生成 + 動作分析 — 從照片/影片/文字/量測值生成精確 3D 人體和動畫。

⚠️ 重要：2026 年 2 月被 Epic Games 收購，4 月 18 日線上平台已關閉。 技術將整合至 Unreal Engine / Epic 數位人類生態系。

解決的問題（歷史）：

  • 學術 SMPL 模型缺乏商業授權通道
  • 從影片提取精確 3D 人體形態+動作
  • 數位人類/Avatar 生成需要統一標準

四、使用流程

⚠️ 平台已於 2026/4/18 關閉。以下為歷史記錄。

MoCapade（動捕）

  1. 上傳影片（任何攝影機：手機/webcam/專業）
  2. AI 提取全身+手指動作
  3. 攝影機追蹤（靜態或移動）
  4. 匯出 FBX/GLB

ME 平台（Avatar 生成）

  1. 上傳照片/影片/量測值
  2. AI 估算 3D 人體形態
  3. 自訂調整
  4. 匯出 FBX/GLB/SMPL

API

  1. 取得 API Key
  2. RESTful 呼叫（JSON:API）
  3. 建立/編輯 Avatar
  4. 下載 3D 資產

六、優缺點分析

✅ 優點（歷史評價）

  1. SMPL 發明者 — 學術根基最深，模型是業界標準（被 100+ 公司/大學使用）
  2. 全身+手指+攝影機追蹤 — 功能完整
  3. 多輸入模態 — 照片/影片/量測/掃描/文字都能生成 3D 人體
  4. 企業級合規 — GDPR + SOC2，適合大型企業
  5. API 可自動化 — RESTful API 支援批次處理
  6. Epic Games 收購 — 證明技術價值（估計收購價數千萬至億級）
  7. 匯出格式完整 — FBX/GLB/SMPL

❌ 缺點

  1. ⚠️ 平台已關閉 — 2026/4/18 起無法使用
  2. SMPL 授權昂貴 — 商業用 €5,000+/年
  3. 技術被 Epic 鎖定 — 未來可能只在 Unreal 生態系使用
  4. 學術導向 — UI/UX 曾不如商業工具友善
  5. 不再獨立服務 — 無法單獨購買/使用

  5. 記錄歷史 — Meshcapade 的收購證明 AI Mocap + 3D 人體技術的商業價值極高（被遊戲引擎巨頭收購）

附錄：為什麼 Meshcapade 重要？

即使平台已關閉，理解 Meshcapade/SMPL 對我們小組仍有價值：

  1. SMPL 是底層標準 — 幾乎所有 AI 動捕工具（包括我們用的 Kimodo）都在某種程度上基於或受 SMPL 影響
  2. Epic 收購的意義 — 說明 AI 人體建模+動捕技術正在從學術走向遊戲引擎內建功能
  3. 未來趨勢 — UE5/6 可能原生支援類似 Meshcapade 的 AI 動捕功能，屆時我們的工作流可能大幅簡化
  4. SMPL 授權問題 — 如果我們的工具鏈中有使用 SMPL 衍生技術的商業工具，需要注意授權合規

十一、後續發展 — 技術已整合進 UE 5.8（2026/6 月更新）

從「付費第三方平台」→「引擎免費內建功能」

2026 年 6 月，Epic Games 在 Unreal Engine 5.8 中正式推出 MetaHuman Animator Markerless Motion Capture Plugin，該功能直接基於 Meshcapade/SMPL 技術：

  • 免費：從 Fab 市集下載，不需任何額外付費
  • 全身+臉+手：單鏡頭影片即可完成所有追蹤
  • 引擎內建：無需外部工具、無需 retarget，直接輸出到 MetaHuman 骨架
  • 本地處理：離線批次處理，不依賴雲端

> 來源確認：80.lv 報導明確指出「This capability comes from integrating Meshcapade's technology」
