MocapNET

即時 3D 姿態估計

由希臘 FORTH 研究院開發的即時神經網路 3D 姿態估計系統。最大賣點：直接輸出 BVH 格式！與 Kimodo Pipeline 完美相容，免費開源。

直接輸出 BVH免費開源即時推斷IK 精修

核心能力

MocapNET 是學術研究院開發的即時 3D 姿態估計系統，最大特色是直接輸出 BVH 格式——這讓它與 Kimodo 的 BVH 輸出完美共用同一套後處理 pipeline。

⚠️ 一般 AI 動捕的格式問題

  • 格式不直接 — 多數工具輸出 JSON/NPZ 等中間格式需轉換
  • 需額外步驟 — SMPL → BVH 的轉換需要額外腳本
  • 精度損失 — 格式轉換過程可能損失精度
  • Pipeline 複雜 — 多了轉換步驟就多了出錯的環節

✅ MocapNET 方案

  • 直接 BVH 輸出 — 原生輸出 BVH，零轉換直接可用
  • 與 Kimodo 共用 — 同是 BVH 格式，共用 Retarget/Foot Fix pipeline
  • 即時處理 — 可即時從攝影機提取 3D 動作
  • IK 精修 — 內建 Inverse Kinematics 求解器修正比例

功能特色

核心能力
  • 直接 BVH 輸出 — 無需格式轉換即可使用
  • 上下半身分離 — 分別估計提高精度
  • IK 求解器 — 內建反向動力學精修關節角度
  • 即時處理 — 可達到接近即時的推斷速度
技術細節
  • 輸入 — OpenPose 2D 關鍵點座標
  • 神經網路 — 多層全連接網路推斷 3D 角度
  • 輸出骨架 — 標準 BVH 骨架定義
  • 開源 — GitHub 完整程式碼

優勢分析

  • 直接 BVH 輸出 — 與所有動畫軟體和 Kimodo 完美相容，零轉換成本
  • 免費開源 — GitHub 上完整程式碼，可自由修改與整合
  • 上下半身分離估計 — 提高各部位的估計精度
  • IK 求解器精修 — 內建的反向動力學確保骨架物理合理性
  • 輕量化 — 相比其他方案，計算需求相對較低

限制與缺點

  • 需要 Linux — 主要支援 Ubuntu，Windows 支援有限
  • 依賴 OpenPose — 需先安裝配置 OpenPose（授權與安裝皆複雜）
  • 安裝複雜 — 多個依賴套件，編譯過程容易出錯
  • 品質不如商業方案 — 精度與穩定性低於付費動捕軟體
  • 維護頻率低 — 學術項目，更新不頻繁

與 Kimodo 的整合可能

極高相容性

MocapNET 直接輸出 BVH 格式，與 Kimodo 使用完全相同的動作表示。兩者共用整個後處理 pipeline——動作平滑、retarget、品質檢查等步驟可直接套用。

整合流程

影片 → OpenPose → MocapNET → BVH 輸出 → 直接進入 Kimodo 後處理 pipeline → 最終產出。無需任何格式轉換步驟。

適用場景

快速動作擷取

從影片快速生成 BVH，作為動畫基礎或參考。

批量處理

大量影片自動化轉換為 BVH 動作資料庫。

研究開發

可修改原始碼，客製化輸出骨架與後處理。
