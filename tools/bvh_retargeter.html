BVH Retargeter — 深度調研報告

工具概述

BVH Retargeter 是一款免費的 Blender 外掛，前身為 MakeHuman 團隊的 MakeWalk 外掛，由開發者 Thomas Larsson (Diffeomorphic) 重新開發為獨立插件。其核心用途為：將 BVH（及 FBX）動畫檔案的動作重新定向（retarget）到目標骨架上，並提供多種動畫編輯與清理功能。

解決的問題：

  • 將 Mocap / AI 生成的 BVH 動作快速套用到不同骨架結構的角色上
  • 不同骨架命名/結構間的骨骼映射（bone mapping）
  • 載入後的動畫編輯（循環、簡化、地板修正等）

功能詳細說明

核心功能

功能說明
Load And Retarget一鍵載入 BVH/FBX 檔案並重定向到目標骨架
骨骼映射（Bone Mapping）支援自定義 JSON 映射檔，內建多種已知骨架預設（Mixamo、Rigify、DAZ、Auto-Rig Pro、VRoid、MMD、Unreal 等）
FBX 支援透過 Blender 內建 FBX Importer 中轉，支援 Mixamo FBX 動畫匯入
Animation Layers使用 NLA 動畫層做非破壞性動畫編輯（2025 年新增）
Loop Animation將動畫轉換為可循環的動作片段
Simplify FCurves簡化動畫曲線，減少關鍵幀數量
Action 管理載入多個動作、切換、重命名
T-Pose 設定自訂源/目標骨架的 T-Pose 參考
Mute/Unmute 骨骼選擇性靜音特定骨骼的動畫
FACS 表情匯入配合 DAZ Importer 匯入面部動畫

使用流程（三步驟）

  1. 選取目標骨架（Armature）
  2. 按下 「Load And Retarget」按鈕
  3. 在檔案選擇器中選取 BVH/FBX 檔案 → 動畫自動載入完成

安裝/使用流程

安裝方式

方法一：Blender Extensions（4.2+）

  • Blender Extensions 平台搜尋 「Retarget」或從 GitHub 下載 zip
  • Edit → Preferences → Add-ons → Install → 選取 zip

方法二：手動安裝

  1. 從 Bitbucket/GitHub 下載最新 zip：
  • GitHub: https://github.com/Diffeomorphic/retarget_bvh
  • Bitbucket: https://bitbucket.org/Diffeomorphic/retarget_bvh/downloads/
  1. Blender → Edit → Preferences → Add-ons → Install → 選取 zip
  2. 啟用外掛

使用流程

  1. 準備目標角色：確保角色有正確的骨架（Rigify、DAZ、自訂皆可）
  2. 選取骨架：在 3D Viewport 中選取目標 Armature
  3. 開啟側邊欄：N 面板 → BVH Retargeter tab
  4. 設定 Source rig：選擇源骨架類型（Mixamo / CMU / 自訂 JSON）
  5. Load And Retarget：選取 BVH/FBX 檔案
  6. 後處理（可選）：Loop、Simplify、Animation Layers 編輯

優缺點分析

✅ 優點

  1. 完全免費開源 — 無任何費用門檻
  2. BVH 原生支援 — 專門針對 BVH 格式設計，與我們的 Kimodo BVH 輸出完美契合
  3. 內建多骨架預設 — Mixamo、Rigify、DAZ、ARP、VRoid、MMD、Unreal 等常見骨架開箱即用
  4. FBX 額外支援 — 可匯入 Mixamo FBX 格式動畫
  5. 動畫編輯工具整合 — Loop、Simplify、Animation Layers 提供基本的 Mocap clean-up
  6. 可程式化/批次操作 — 支援 Python scripting，可自動化批次 retarget
  7. 長期維護 — 從 MakeWalk 時代持續更新超過 10 年
  8. DAZ Importer 生態整合 — 與同作者的 DAZ Importer 無縫配合

❌ 缺點

  1. 無 GUI 視覺化骨骼映射 — 自訂映射需手動編寫 JSON，學習門檻較高
  2. 沒有腳部 IK/接地修正 — 不處理滑步、穿地等問題（需搭配其他工具）
  3. 文件較分散 — 主要靠 blog post，無統一官方文件站
  4. 無商業支援 — 個人開發者維護，bug 回應不一定即時
  5. 不處理 Root Motion — 缺乏 root motion 提取/轉移功能
  6. 社群較小 — 相比 Auto-Rig Pro 等付費工具，使用者基數較少

與 Kimodo BVH 輸出的整合可能性

整合評估：⭐⭐⭐⭐⭐（高度適合）

面向評估
格式相容性✅ Kimodo 輸出 BVH → BVH Retargeter 原生支援
骨架映射⚠️ 需建立 Kimodo 專用的 source rig JSON 映射檔（一次性工作）
批次處理✅ 支援 Python scripting，可寫批次 retarget 腳本
自動化整合✅ 可嵌入我們的 pipeline（Blender headless + Python）
品質⚠️ retarget 品質中上，但缺乏腳部修正（需 Foot Locker/In Placer 補足）

建議整合流程


Kimodo BVH → BVH Retargeter（retarget 到遊戲骨架）
    → Foot Locker/In Placer（修腳步接地）
    → Export FBX/GLB（匯入遊戲引擎）

需要的一次性準備工作

  1. 分析 Kimodo 輸出 BVH 的骨架結構與命名
  2. 建立 kimodo_source.json 骨骼映射檔
  3. 建立 Blender Python 批次腳本模板
  4. 測試常見動作類型的 retarget 品質

對「AI 動作量產工作流」的貢獻定位

定位：pipeline 第一環 — BVH 轉遊戲骨架


[Kimodo AI 生成 BVH]
        ↓
[BVH Retargeter] ← 本工具定位
  - retarget 到統一遊戲骨架
  - 基本動畫清理（simplify、loop）
  - 批次自動化處理
        ↓
[Mocap Clean-up 工具鏈]
  - Foot Locker（接地修正）
  - In Placer（原地化）
  - Root Motion Transfer
        ↓
[遊戲引擎匯入]

核心價值：

  • 免費、可程式化、BVH 原生支援
  • 作為 AI 動作量產 pipeline 的「格式轉換+骨架適配」第一站
  • 搭配其他工具形成完整 clean-up 鏈路

限制：

  • 不解決滑步/穿地（交給專用腳步工具）
  • 不處理 root motion 問題（交給 Root Motion Transfer）
  • 需要一次性建立 Kimodo 骨架映射
