ReMoCapp — 深度調研報告

七、Pipeline 整合分析

項目說明
輸入來源多攝影機同步影片
輸出格式FBX、BVH
骨架標準標準 Humanoid
上游工具多台攝影機（手機/Webcam）
下游工具Blender、Maya、UE5、Unity

推薦搭配：ReMoCapp 多攝影機 → BVH/FBX → Blender retarget → 遊戲引擎。多攝影機方案的另一個選擇，品質優於單鏡頭但設置較複雜。

推薦整合方式


高品質需求（有演員）：
[演員表演] → [ReMoCapp 即時動捕] → [BVH] → [ARP Remap] → [引擎]

量產需求（無演員）：
[Prompt] → [Kimodo] → [BVH] → [ARP Remap] → [引擎]

混合使用：
[ReMoCapp 錄製參考] → [觀察動作] → [撰寫 Kimodo Prompt] → [量產]

ReMoCapp 適合「有演員、需要即時和臉部」的場景；Kimodo 適合「批次量產、無演員」的場景。

8. 對 AI 動作量產工作流的貢獻定位

定位：即時動捕的中端方案（有演員時使用）


高端：[OptiTrack/Vicon] → [$$$$$] → [最高精度]
中端：[ReMoCapp] → [$$] → [中高精度 + 即時 + 臉部]
低端：[TDPT/單攝影機] → [免費] → [基礎品質]
AI 端：[Kimodo] → [免費] → [中高品質，無需演員]

量產流程中的角色：

  • 特殊動作錄製（需要真人表演參考時）
  • 臉部動捕來源（ARKit BS 可直接用於 FaceIt 系統）
  • 即時預視和互動場景
  • 非日常工具，是「需要時才用」的特殊工具

10. 行動建議

  1. 評估空間和預算：是否有固定空間可設置 4+ 攝影機
  2. 試用/Demo：聯繫官方取得試用版本
  3. 定位為特殊工具：非日常使用，當需要真人參考動作時使用
  4. 重點評估臉部品質：ReMoCapp 的臉部動捕可搭配 FaceIt 使用
  5. 與 Kimodo 明確分工：ReMoCapp 做「有演員的」，Kimodo 做「無演員的」
