一、工具概述

AGI Mocap（Artificial Graphic Intelligence）是 DiMocap 的最新 AI 動捕演算法，專門針對「AI Mocap 常見品質問題」設計。最大特色：強力抗遮擋（半身遮擋可捕）+ 防滑步/穿地/飄浮。

核心能力一句話：Webcam 影片 → AI 動捕 → 自動解決滑步/穿地/遮擋問題 → BVH/FBX/BIP 匯出。

解決的問題：

  • AI 動捕常見的「腳滑步」問題
  • 動作穿過地面
  • 身體飄浮不自然
  • 遮擋造成追蹤失敗（AGI 半身遮擋也能追蹤）

六、優缺點分析

✅ 優點

  1. BVH 直接匯出 — 與 Kimodo 格式完全相容
  2. 強力抗遮擋 — 獨特優勢，半身遮擋也能追蹤
  3. 自動防滑步/穿地 — 內建品質修正
  4. 低成本 — 定位親民
  5. 多格式匯出 — BVH/FBX/BIP/VMD

❌ 缺點

  1. 英文資訊有限 — 中國產品
  2. 無 API/批次 — 手動操作
  3. 無臉部/手指 — 僅全身
  4. 品質未獨立驗證 — 無第三方對照測試
  5. 軟體取得需聯繫 — 非自助下載

七、Pipeline 整合分析

項目說明
輸入來源Webcam / 影片檔案（強力抗遮擋）
輸出格式BVH、FBX、BIP、VMD
骨架標準標準骨架
上游工具任何影片（特別適合遮擋嚴重的影片）
下游工具Blender（BVH 匯入）、3DS Max（BIP）、Maya（FBX）

推薦搭配：遮擋嚴重的動作參考影片 → AGI Mocap（抗遮擋演算法）→ BVH → Blender retarget。其他 AI Mocap 追蹤失敗時的備選方案。

八、對「AI 動作量產工作流」的貢獻定位


AGI Mocap 定位：「特殊場景的品質修正方案」

適合：
• 遮擋嚴重的動作參考影片
• 需要 BVH 格式的直接匯出
• 低預算本地動捕

不適合：
• 大規模自動化量產
• 需要臉部/手指
• 需要 API
