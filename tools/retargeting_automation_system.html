Retargeting Automation System — 深度調研報告

工具概述

Retargeting Automation System 是一款付費的 Blender 外掛，透過 Superhive Market (原 Blender Market) 販售。定位為自動化的 Mocap Retargeting 方案，旨在簡化並自動化 motion capture 動畫的 retarget 流程。

根據 80.lv 近期報導（2026年6月），由 Pro Constantinou ("Good Good") 開發的 retargeting 工具強調了一個核心理念：更好的 retarget 品質來自於在 retarget 前先匹配來源/目標骨架的 hip 位置和肢體比例。

解決的問題：

  • 傳統 retarget 流程繁瑣且需大量手動調整
  • 來源/目標骨架比例差異導致的腳步滑動和彈跳
  • 需要自動化的批次 retarget pipeline
  • Hip 位置和肢體比例不匹配造成的品質問題

核心理念（來自開發者說明）

開發者 Pro Constantinou 的方法論：

> 「更好 mocap retarget 的關鍵是在匹配其餘姿勢之前，先將目標骨架的 hip 位置和肢體比例與來源匹配。Hip 與腿/脊椎之間最微小的相對位置差異都會造成嚴重的肢體滑動或彈跳，讓角色無法感覺腳踏實地。」

方法論步驟：

  1. 縮放來源骨架匹配目標角色 → 但這只能到一定程度
  2. 對齊 Hip 骨骼 → 性價比最高的調整
  3. 縮放肢體匹配關節位置 → 犧牲些許拉伸/壓縮換取更好的關節對齊
  4. 最終達到更精確的 retarget 結果

安裝/使用流程

安裝

  1. 從 Superhive Market 購買並下載
  2. Blender → Edit → Preferences → Add-ons → Install → 選取 zip
  3. 啟用外掛

使用流程（推測）

  1. 匯入來源動畫（BVH/FBX）
  2. 指定目標角色骨架
  3. 工具自動分析比例差異
  4. 自動對齊 Hip + 肢體比例
  5. 執行 Retarget
  6. 結果直接具有更好的接地品質

優缺點分析

✅ 優點

  1. 源頭解決滑步 — 不是事後修補，而是從比例匹配入手預防滑動
  2. 自動化流程 — 減少手動設定步驟
  3. 理念先進 — Hip alignment + limb scaling 是有根據的方法
  4. 接地品質好 — 開發者特別強調 ground contact 效果
  5. 新開發 — 融合最新技術理解
  6. 80.lv 報導 — 有業界媒體認可

❌ 缺點

  1. 付費 — 增加成本
  2. 較新 — 2026 年發佈，社群驗證不多
  3. 公開資訊有限 — 付費工具，詳細功能需購買後才知
  4. 可能的肢體拉伸 — 縮放肢體匹配會導致些許 stretching/squashing
  5. 未知批次能力 — 不確定是否支援大量動畫的自動化批次處理
  6. 不確定 BVH 直接支援 — 可能需要先匯入 FBX

  • 從 retarget 方法論上提升品質（非事後修補）
  • 可能減少腳步後處理工具的需求

不確定因素：

  • 批次處理能力（量產關鍵）
