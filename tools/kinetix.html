Kinetix — 深度調研報告

1. 工具概述

Kinetix 是一家法國 AI 動畫公司，提供 SDK 讓遊戲開發者在遊戲中嵌入 AI 動畫生成功能。玩家可透過上傳影片或文字描述，在遊戲內即時生成自訂 3D 表情動作（emotes）。定位為 B2B 遊戲開發平台，專注於玩家 UGC 動畫生態。

核心能力一句話：讓遊戲玩家在遊戲內用影片/文字即時生成自訂動畫的 B2B SDK 平台。

解決的問題：

  • 遊戲中的 emote/表情動作選擇有限
  • 玩家 UGC 內容（用戶生成內容）僅限於文字/圖片，缺乏動畫創作工具
  • 遊戲開發者缺乏現成的「遊戲內 AI 動畫生成」方案
  • 跨遊戲動畫資產不相容

4. 安裝/使用流程

對遊戲開發者（B2B）

  1. 前往 https://www.kinetix.tech/ 申請開發者帳號
  2. 取得 API Key 和 SDK 存取權
  3. 下載 Unity SDK 或 Unreal SDK
  4. 整合到遊戲專案中：
  • 加入 SDK package
  • 配置 API Key
  • 呼叫 API 端點
  1. 在遊戲 UI 中設計動畫生成入口
  2. 發佈遊戲——玩家即可使用

對玩家端（使用流程）

  1. 在支援 Kinetix 的遊戲中找到「創建動作」入口
  2. 選擇輸入方式：上傳影片 / 輸入文字描述
  3. AI 雲端處理生成動畫
  4. 在遊戲內預覽效果
  5. 確認後套用到角色

注意：此為 B2B 工具，個人動畫師/開發者無法直接當作動捕工具使用。

6. 優缺點分析

優點

  • ✅ 遊戲內原生整合：玩家無需離開遊戲即可創建動畫
  • ✅ Text-to-Animation：文字描述直接生成，降低創作門檻
  • ✅ 多引擎支援：Unity + Unreal + Web
  • ✅ UGC 生態：玩家創建的動畫可交易，增加遊戲生態
  • ✅ 雲端處理：無需客戶端算力
  • ✅ 內容審核：避免不當內容

缺點

  • ❌ B2B Only：個人動畫師無法直接使用
  • ❌ 黑盒子：AI 模型細節不透明
  • ❌ 依賴雲端：無離線模式，需持續付費
  • ❌ 品質不確定：AI 生成品質可能參差不齊
  • ❌ 價格不透明：需個案洽談
  • ❌ 非 BVH 工具：不是傳統動捕/動畫製作工具
  • ❌ 玩家動畫品質受限：非專業人士上傳的影片品質可能不佳

推薦整合方式

Kinetix 和 AI 動作生成工具 不是同一賽道：

  • Kinetix = 遊戲內玩家 UGC
  • AI 動作生成工具 = 開發期間的動畫量產

開發階段：[AI 動作生成工具] → [BVH] → [ARP] → [引擎]（角色預設動作庫）
營運階段：[Kinetix SDK] → [遊戲內玩家 UGC 動畫]（玩家自訂 emotes）

兩者可在同一遊戲中共存：AI 動作生成工具 做基礎動作庫，Kinetix 做玩家 UGC 層。

8. 對 AI 動作量產工作流的貢獻定位

定位：遊戲營運期的玩家 UGC 動畫平台（非開發期工具）


┌─── 開發期（我們的工作）──────────────────┐
│ [AI 動作生成工具] → [BVH] → [ARP] → [引擎動作庫]  │
└────────────────────────────────────────────┘

┌─── 營運期（遊戲上線後）──────────────────┐
│ [Kinetix SDK] → [玩家 UGC 動畫生態]       │
└────────────────────────────────────────────┘

量產流程中的角色：

  • 不在「動作設計師」的核心工作流中
  • 是給「遊戲設計師/製作人」的營運工具建議
  • 如果我們的遊戲需要玩家 UGC 動畫功能，Kinetix 是考慮選項
