解決什麼問題？

傳統 3D 動畫耗時耗力，而 AI 影片生成（如 Sora）輸出不可編輯的像素。Cartwheel 填補了這個空缺。

❌ 傳統方式

  • 高門檻 — 3D 動畫需要數年學習
  • 不可編輯 — AI 影片生成輸出像素無法修改
  • 供不應求 — 動作資料需求遠超供給

✅ Cartwheel 方案

  • 多模態輸入 — 文字/影片/Pose/Library 四種方式
  • 可編輯輸出 — Rig 控制資料，DCC 工具完全編輯
  • 乾淨曲線 — 已清理的動畫曲線（非逐幀 Mocap key）

優缺點分析

基於實際調研的綜合評估。

✅ 優點

  • 輸出可編輯 Rig 資料（核心差異化）
  • 瀏覽器零安裝
  • 多模態輸入（text/video/pose）
  • 前 OpenAI+Google 團隊
  • 免費方案可用
  • 不訓練使用者資料

❌ 缺點

  • 生成品質尚不穩定
  • 不確定 BVH 匯出
  • Spin 額度限制
  • 無離線方案
  • 相對較新（2025/5 公開）

與 Kimodo 的整合方式

Cartwheel 與 Kimodo 互補：Kimodo 離線 text-to-motion，Cartwheel 雲端多模態。

  • 格式 — Kimodo=BVH, Cartwheel=GLB/FBX — 都是標準格式可在 Blender 統一處理
  • 互補 — Kimodo 離線本地 vs Cartwheel 雲端多模態
  • Video Ref — Cartwheel 的 Video-to-Motion 補充 Kimodo 沒有的能力
  • API — Cartwheel API 開發中，未來可自動化

量產工作流定位

Cartwheel 在量產工作流中擔任「快速原型+影片參考」角色。

快速原型 → Cartwheel（text/video/pose） 正式量產 → Kimodo（離線 text-to-motion） 兩者統一 → Blender retarget → 遊戲引擎
