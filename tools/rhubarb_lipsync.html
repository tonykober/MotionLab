Rhubarb Lipsync — 深度調研報告

工具概述

Rhubarb Lip Sync 是一個開源命令列工具，由 Daniel S. Wolf 開發，能從語音錄音自動產生 2D 嘴型動畫資料。它分析音訊檔案、辨識語音內容，並自動生成 lip sync 時間軸，標示每個時間點應顯示哪種嘴型。可用於遊戲角色對話、動畫卡通等任何需要口型同步的專案。

解決的問題：

  • 手動對嘴型（lip sync）極度耗時
  • 需要自動化的語音→嘴型映射
  • 需要跨平台（Windows/macOS/Linux）且可整合到自動化流程的工具
  • 需要支援多種輸出格式以整合不同軟體

功能詳細說明

嘴型系統

6 個基礎嘴型（Hanna-Barbera 經典標準）+ 3 個擴展嘴型：

嘴型用途
Ⓐ閉嘴（P、B、M 音）
Ⓑ微開齒關（大多數子音 + EE 音）
Ⓒ開口（EH、AE 音 + 部分子音）
Ⓓ大開口（AA 音，如 father）
Ⓔ微圓口（AO、ER 音）
Ⓕ嘟嘴（UW、OW、W 音）
Ⓖ（擴展）上齒咬下唇（F、V 音）
Ⓗ（擴展）舌頭上抬（長 L 音）
Ⓧ（擴展）靜止位置（說話間暫停）

語音辨識引擎

引擎適用
PocketSphinx英語錄音（預設，品質較好）
Phonetic非英語錄音（語言無關，品質稍差）

輸出格式

格式說明
TSV最簡潔，時間戳 + 嘴型代碼
XML含 metadata（檔案路徑、時長）
JSON結構化，易程式解析
DATMoho/OpenToonz 專用

安裝/使用流程

安裝（核心工具）

  1. 從 GitHub Releases 下載對應平台的二進位檔（Windows/macOS/Linux）
  2. 解壓到任意位置
  3. 命令列即可使用

安裝（Blender 整合）

  1. 下載 Blender 外掛 zip（最新版已內建 Rhubarb 執行檔）
  2. Blender → Preferences → Add-ons → Install
  3. 設定 Rhubarb 執行檔路徑（若未內建）

使用流程


# 基本用法
rhubarb -o output.json my-recording.wav

# 帶文字稿（提升精確度）
rhubarb -d script.txt -o output.json my-recording.wav

# 非英語（使用 phonetic 辨識器）
rhubarb -r phonetic -o output.json my-recording.wav

# 指定輸出格式
rhubarb -f json -o output.json my-recording.wav

Blender 工作流


1. 為角色建立 Pose Library（含 A-X 嘴型 Poses）
2. 匯入音訊到 Blender
3. 執行 Rhubarb 外掛 → 選擇音訊檔
4. 自動在時間軸上生成嘴型 Keyframes
5. 微調結果

優缺點分析

優點

  • ✅ 完全免費開源：MIT 授權，商業可用
  • ✅ 跨平台：Windows / macOS / Linux
  • ✅ 命令列介面：完美適合自動化批次處理
  • ✅ 業界標準嘴型：Hanna-Barbera 6 嘴型系統 = 動畫業界標準
  • ✅ 多輸出格式：TSV / XML / JSON / DAT
  • ✅ 對白文本輔助：提供文字稿可大幅提升品質
  • ✅ Machine Readable：自動化整合友好
  • ✅ 多軟體整合：After Effects / Spine / Moho / Blender 等
  • ✅ GitHub 2.5k+ Stars：成熟穩定

缺點

  • ❌ 僅英語最佳：PocketSphinx 僅支援英語，其他語言用 Phonetic 精度下降
  • ❌ 非中文友好：中文語音辨識品質未知/可能很差
  • ❌ 命令列操作：需技術基礎，非 GUI 友好
  • ❌ 2D 嘴型思維：原始設計為 2D 動畫，3D 角色需額外映射
  • ❌ 不處理 3D Shape Keys 映射：需透過 Blender 外掛橋接
  • ❌ 最後更新 2021：核心工具開發較不活躍

身體 + 口型Kimodo 生成身體動作，Rhubarb 處理口型同步

與 FaceIt 的互補

FaceItRhubarb

建議整合流程


語音錄音 → Rhubarb 分析 → 嘴型時間軸（JSON）
→ Blender 外掛映射到 FaceIt 的 Shape Keys
→ 合併 Kimodo 身體動作 → ARP 匯出 → 遊戲引擎

對「AI 動作量產工作流」的貢獻定位

定位：語音→嘴型的自動化橋樑


[TTS/錄音] → [Rhubarb Lip Sync] → [嘴型時間軸] → [Blender 映射] → [完整臉部動畫]
                                                        ↑
                                               FaceIt Shape Keys

具體貢獻：

  1. 語音資產批次處理：遊戲有數百條對話，Rhubarb 可全自動化處理
  2. TTS 整合：AI 生成語音（TTS）→ Rhubarb 自動口型 = 完全自動化
  3. 免費且可嵌入：MIT 授權可嵌入任何工具鏈
  4. 命令列 = 可腳本化：完美適合量產管線

限制：

  • 中文支援不佳（僅 Phonetic 模式）
  • 精度不如手動對口型
  • 需要 FaceIt 或類似工具提供 Shape Keys 資產
