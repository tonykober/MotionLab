Marionette Mocap — 本地端 AI 動作捕捉（含身體/臉/手）

調研日期：2026-06-19

官網：https://marionettemocap.com

定價：$12.50/月起（無限渲染）

插件：Blender / Unity / Maya

一、工具概述

Marionette Mocap 是一款本地端 AI 動作捕捉工具，支援全身、臉部和手部追蹤。核心差異化：本地處理（隱私+離線）+ 無限渲染（不按秒計費）+ DCC 插件即時串流。

核心能力一句話：從影片/webcam 提取身體+臉+手動畫 → 本地無限處理 → 即時串流到 Blender/Unity/Maya。

解決的問題：

  • 雲端 Mocap 方案（Viggle/DeepMotion）按秒/按量計費，量大成本高
  • 隱私敏感項目不能上傳到雲端
  • 傳統工作流需匯出→匯入，Marionette 直接串流到 DCC

六、優缺點分析

✅ 優點

  1. **無限使用不按量計費** — 月費固定，不管處理多少影片
  2. **本地處理離線可用** — 隱私安全，不需網路
  3. **身+臉+手全覆蓋** — 一個工具搞定全部
  4. **DCC 即時串流** — 不需匯出/匯入，直接在 Blender/Unity 預覽
  5. **價格低** — $12.50/月 vs Move.AI/Plask 更貴
  6. **免費插件** — Blender/Unity/Maya 插件免費
  7. **智慧清理** — 內建自動修正品質

❌ 缺點

  1. **無 BVH 匯出** — 只有 FBX（Mixamo），與 Kimodo BVH 需轉換
  2. **無 API/批次** — 手動操作，不適合全自動化
  3. **較新工具** — 市場認知度低，獨立品質驗證少
  4. **精度未正式對比** — 未見與 Vicon/OptiTrack 的對照測試
  5. **需要本地 GPU** — 本地處理需算力

七、與 Kimodo BVH 工作流的整合可能性

面向評估
**格式**Kimodo=BVH，Marionette=FBX(Mixamo) — 需在 Blender 轉換但 Mixamo 標準很好處理
**互補場景**Kimodo=text-to-motion；Marionette=video-to-motion(含臉手)
**量產化**Marionette 無限但手動；Kimodo 離線可腳本化
**臉/手**Marionette 補充 Kimodo 沒有的臉部和手指
**成本**兩者都極低（$12.50/月 vs Kimodo 本地免費）

推薦整合方式


方案：混合工作流
  全身表演動作 → Marionette (含臉+手) → FBX → Blender retarget
  文字描述動作 → Kimodo → BVH → Blender retarget
  統一在 Blender 合併 → 匯出到遊戲引擎
