Root Motion Batch Transfer — 深度調研報告

工具概述

Root Motion Batch Transfer 是一款免費開源的 Blender 外掛，由開發者 NguyenNP-24 開發。其核心功能是將角色的位移動畫（原本記錄在 Hips/COG 等控制器上）轉移到 Root 骨骼上，這是遊戲引擎中 Root Motion 系統所需的標準格式。支援批次處理多個動作。

解決的問題：

  • Mocap / AI 動畫的位移通常記錄在 Hips 骨骼上，但遊戲引擎 Root Motion 需要在獨立的 Root Bone 上
  • 手動轉移 Root Motion 是繁瑣的重複工作
  • 需要批次處理大量動畫的 Root Motion 提取

功能詳細說明

核心功能

功能說明
Transfer to Root將選取控制器的位移轉移到 Root 骨骼
Batch Transfer to Root一次批次處理所有 Actions 的 Root Motion
Add Controller指定哪些控制器參與 Root Motion 計算（IK 腳/手、Torso/COG）
Root Controller指定 Root 骨骼
Target Controller指定 Root 要跟隨的目標（通常是 Torso/COG）
Keep in World Origin選擇 Root 是否重置到世界原點（0,0,0）
任意 Rig 支援不限骨架類型，任何有 IK 控制器的 rig 都能用

工作原理

  1. 使用者指定所有「移動整個角色」的控制器骨骼
  2. 外掛計算這些控制器的合成位移
  3. 將位移數據 Bake 到 Root Bone 的動畫曲線上
  4. 原控制器的位移被相應抵消

兩種模式

  • Keep in World Origin = ON：Root Motion 轉移後，Root 骨骼回到原點（適合原地動畫 + Root Motion 曲線匯出）
  • Keep in World Origin = OFF：Root 跟隨 Target Controller 移動（適合預覽完整運動）

使用流程

  1. 選取 Armature → 進入 Pose Mode
  2. 選取所有參與整體位移的控制器（IK 腳、IK 手、Torso/COG）
  3. 點擊 Add Controller 加入清單
  4. 指定哪個是 Root Controller
  5. 指定 Target Controller（Root 要跟隨的目標）
  6. 點擊 Transfer to Root（單一動作）或 Batch Transfer to Root（所有動作）

優缺點分析

✅ 優點

  1. 完全免費開源 — GPL-3.0
  2. 批次處理 — 一次處理所有 Actions，適合量產工作流
  3. 任意 Rig 支援 — 不限骨架類型（Rigify、ARP、Mixamo、自訂皆可）
  4. 靈活的控制器系統 — 可指定任意數量的控制器參與計算
  5. 兩種模式 — World Origin / Follow Target 滿足不同需求
  6. 最新版相容 — 支援 Blender 5.1
  7. 有影片教學 — 開發者提供完整使用教學影片
  8. 遊戲引擎導向 — 專為 game-ready 動畫設計

❌ 缺點

  1. 社群較小 — 39 Stars，可能遇到問題時支援有限
  2. 只處理 Root Motion — 不做其他 clean-up（腳步、接地等）
  3. 需要 IK rig — 最適合有 IK 控制器的骨架
  4. 手動指定控制器 — 需使用者判斷哪些骨骼參與位移
  5. 文件較少 — 主要依靠 YouTube 影片
  6. 較新的工具 — 尚未經過大量使用者驗證

核心價值：

  • 解決 AI 動畫「位移在錯誤骨骼上」的問題
  • 批次處理非常適合量產場景
  • 免費開源，可修改整合到自動化 pipeline

何時需要 Root Motion：

  • 動畫需要精確位移控制（格鬥、閃避、翻滾等）
  • 角色移動由動畫驅動（非程式碼速度控制）
  • 需要腳步與地面完美同步

何時不需要（用 In Place 代替）：

  • 簡單的跑步/走路循環（程式碼控制移動速度）
  • 角色移動由物理系統驅動
