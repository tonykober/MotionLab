七、Pipeline 整合分析

項目說明
輸入來源多攝影機同步影片（multi-camera 方案）
輸出格式FBX、BVH（推測標準格式）
骨架標準標準骨架
上游工具多台同步攝影機
下游工具Blender、Maya、UE5、Unity

推薦搭配：mimem AI 多攝影機 → 高精度 3D 重建 → FBX/BVH → DCC 工具。多攝影機方案品質高於單攝影機 AI，適合需要更高精度的場景（介於 Viggle 和 Move.AI Genesis 之間）。

推薦整合方式

[演員表演] → [mimem 多角度拍攝] → [BVH] → [ARP] → [引擎]

場景 B（無法實拍）：
[Prompt] → [Kimodo] → [BVH] → [ARP] → [引擎]

mimem 和 Kimodo 分別解決「有影片素材」和「無影片素材」的動作來源問題。

8. 對 AI 動作量產工作流的貢獻定位

定位：低成本多攝影機動捕方案（等待驗證）


傳統高端路線：[OptiTrack/Vicon] → [$$$] → [高精度 BVH]
mimem 路線：  [手機 × N] → [雲端 AI] → [中高精度 BVH]
Kimodo 路線： [文字 Prompt] → [AI 生成] → [BVH]

量產流程中的角色：

  • 目前：觀望（Early Access）
  • 未來：若品質確認夠好，可成為「有參考影片時」的動捕方案
  • 與 Kimodo 互補：Kimodo 做想像的動作，mimem 做實拍的動作
  • 關鍵觀察：免校準的精度是否真的足夠

10. 行動建議

  1. 申請 Early Access：取得測試資格，第一時間評估品質
  2. 準備測試場景：設計標準動作（walk/run/jump）用多手機拍攝
  3. 品質對比：mimem 輸出 vs Kimodo 輸出 vs 手動製作
  4. 評估免校準精度：這是核心賣點，需實測驗證
  5. 追蹤定價公告：正式上線後評估成本效益
