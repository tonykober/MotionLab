一、工具概述

QuickMagic 是 AI 動捕工具，核心特色是自動關鍵幀偵測和多格式匯出（FBX/Mixamo/BIP/UE/VMD）。2025 年 10 月與 Reallusion 合作推出 iClone Video Mocap 服務。

核心能力一句話：上傳影片 → AI 自動偵測 keyframe → 匯出 FBX/BIP/UE 格式 → 3DS Max/Maya/MotionBuilder 直接使用。

解決的問題：

  • 影片轉動畫後需要大量手動修 keyframe
  • 不同軟體需要不同格式（FBX/BIP/UE 各有適用場景）
  • 3DS Max 用戶缺乏好用的 AI Mocap 方案

四、使用流程

  1. 註冊 → https://www.quickmagic.ai/register
  2. 上傳影片 → 瀏覽器介面上傳 MP4
  3. 設定範圍 → 設 in/out 點選取動作片段
  4. AI 處理 → 自動偵測 keyframe + 動捕
  5. 選擇格式 → FBX / Mixamo / BIP / UE / VMD
  6. 下載 → 取得動畫檔案
  7. 匯入 DCC → 3DS Max / Maya / Blender（用 Auto-Rig Pro retarget）

六、優缺點分析

✅ 優點

  1. 多格式匯出 — FBX/Mixamo/BIP/UE/VMD 覆蓋幾乎所有流程
  2. 自動關鍵幀 — 減少後處理手動修 keyframe 工作
  3. 3DS Max 友善 — BIP 格式直接支援（少見）
  4. Reallusion 合作 — iClone 整合+品質背書
  5. 瀏覽器即用 — 無需安裝
  6. tato.studio 評價正面 — 「可靠、預算友好的基礎動畫方案」

❌ 缺點

  1. 無 BVH 匯出 — 與 Kimodo 格式不直接相容
  2. Credit 制 — 量大成本增加
  3. 精度評價中等 — 適合 blocking 不一定適合最終品質
  4. 無 API — 手動操作
  5. 中國平台 — 部分地區可能存取受限

七、與 Kimodo BVH 工作流的整合可能性

面向評估
格式無 BVH — 需要 FBX→BVH 轉換（Blender 可做）
互補性QuickMagic 的 BIP/UE 格式適合直接入引擎，Kimodo 的 BVH 需 retarget
品質QuickMagic 適合快速 blocking，Kimodo 適合特定動作生成
3DS Max 場景如果團隊用 3DS Max，QuickMagic 的 BIP 格式很有價值

推薦整合方式


如果用 3DS Max Pipeline：
  影片參考 → QuickMagic → BIP → 3DS Max 直接使用

如果用 Blender Pipeline：
  影片參考 → QuickMagic → FBX → Blender → Auto-Rig Pro retarget
  文字描述 → Kimodo → BVH → Blender → retarget

八、對「AI 動作量產工作流」的貢獻定位


QuickMagic 定位：「多格式橋樑工具」

最適合：
• 3DS Max / MotionBuilder 用戶（BIP 格式直通）
• iClone / MetaHuman 工作流
• 需要快速 blocking 的前期預覽
• 需要 VMD 格式（MMD）的場景

不適合：
• 大規模自動化（無 API）
• 需要最高品質的最終產出
• Blender-only 工作流（Viggle/Marionette 更直接）
