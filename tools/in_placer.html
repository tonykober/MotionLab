In Placer — 深度調研報告

工具概述

In Placer 是一款免費的 Blender 外掛，由開發者 Fatih Pehlevan 開發（與 Foot Locker 同作者）。其唯一功能是將帶有位移的動畫轉換為「原地動畫」（In-Place Animation），這是遊戲引擎中角色移動系統的標準需求。

解決的問題：

  • 遊戲引擎中角色移動由程式控制（capsule component），動畫本身需要是原地播放
  • Mixamo / AI 生成的動畫通常帶有前進/側移位移，需要移除
  • 手動在 Graph Editor 清除 Root 骨骼位移是重複性工作
  • 一鍵自動化移除前進/側移位移

功能詳細說明

核心功能

功能說明
Make it in place一鍵移除前進+側向移動，保留垂直（跳躍）
One Direction Only只移除單一方向的移動（前進 OR 側向）
Mixamo 支援自動識別 mixamorig:Hips 骨骼
Unreal 格式支援識別 "root" 命名的骨架
UI 位置可選可選擇顯示在 N-Panel 或 Armature Properties
自訂面板名稱Addon Preferences 可修改面板標題

工作原理

  1. 偵測角色的 root/hips 骨骼
  2. 分析該骨骼在動畫中的水平位移 (X/Y 軸)
  3. 從動畫曲線中移除水平位移分量
  4. 保留垂直位移（跳躍/蹲下等需要的 Z 軸）

為什麼需要原地動畫？

在遊戲引擎中：

  • 角色位移由程式碼控制（Capsule Component / Character Controller）
  • 動畫本身只負責「看起來在走/跑」
  • 如果動畫帶有位移，角色會產生雙重移動（程式+動畫）導致滑動

有位移的動畫在遊戲中的問題：

→ 角色位移和動畫位移疊加 → 看起來在「溜冰」或「飄移」

安裝/使用流程

安裝

  1. 從 Gumroad 免費下載：https://fatihpehlevan.gumroad.com/l/InPlacer
  2. Blender → Edit → Preferences → Add-ons → Install → 選取 zip
  3. 啟用 In Placer

使用流程

  1. 確保在 Object Mode
  2. 選取含有動畫的 Armature 物件
  3. N-Panel → In Placer（或 Armature Data Properties）
  4. 點擊 「Make it in place」 → 完成

注意事項

  • Mixamo rig：Root 骨骼名稱必須是 mixamorig:Hips
  • Unreal 格式：Armature 物件名稱必須是 root
  • 場景中不能有同名的 armature（如 root.001 會報錯）
  • 只支援 Mixamo 和 Unreal 格式的骨架命名

優缺點分析

✅ 優點

  1. 完全免費 — 零成本
  2. 一鍵操作 — 學習成本為零，按一個鍵就完成
  3. 節省重複工作 — 大量 Mixamo 動畫不用每次手動清 graph editor
  4. 遊戲開發必備 — 原地動畫是遊戲引擎的基本需求
  5. 保留垂直移動 — 不會破壞跳躍/蹲下等動作
  6. UI 可自訂 — 可隱藏在 N-Panel 或放在 Properties

❌ 缺點

  1. 只支援 Mixamo/Unreal 骨架 — 不支援任意骨架命名
  2. 不處理旋轉 — 原始版本不移除旋轉位移（需 Animazing 處理）
  3. 無批次處理 — 一次只處理一個動畫
  4. 場景命名限制 — 同名 armature 會報錯
  5. 無 Root Motion 提取 — 只移除位移，不另存為 Root Motion 曲線
  6. 功能極度簡單 — 就只有「移除水平位移」這一件事

核心價值：

  • 零成本、一鍵操作的原地化
  • 適合已經是 Mixamo 骨架的情況

限制：

  • 如果已用 ARP Remap，其內建 In Place 功能更強（含旋轉+批次）
  • Kimodo 骨架需先 retarget 才能用
  • 不處理旋轉（轉圈走路會有問題）

結論：

如果工作流已包含 Auto-Rig Pro，In Placer 的價值被 ARP In Place 取代。

如果用免費工具鏈（BVH Retargeter → Mixamo 骨架），In Placer 是最簡單的原地化方案。
