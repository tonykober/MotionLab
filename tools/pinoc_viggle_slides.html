影片 → 3D 骨架動畫 → 遊戲引擎

PINOC 是 Viggle.ai 推出的 3D Studio 功能模組，專門將普通影片轉換為專業級 Motion Capture 資料。Viggle 原本以 AI 影片迷因生成聞名，2026 年正式轉型推出面向遊戲開發者和 3D 創作者的專業工具。

只需要一支手機拍攝的影片，PINOC 就能產出 65 骨骼的 Mixamo 相容骨架動畫，以 GLB/FBX 格式匯出，直接匯入 Unity、Unreal Engine 5、Godot 等遊戲引擎使用。每個動畫成本僅 $0.03。

優缺點分析

PINOC 在成本和易用性上有壓倒性優勢，但在格式支援和特殊場景品質上仍有改進空間。以下是完整評估：

✅ 優點

  • 💰 極低成本 — $0.03/動畫，比傳統 Mocap 便宜萬倍，可以大量試錯不心疼
  • 🆓 零門檻試用 — Guest 模式無需帳號，每日 5 次免費，立即評估品質
  • 🦴 標準骨架 — 65 骨骼 Mixamo 命名直入引擎，省去 retarget 環節
  • 🦶 內建修正 — Foot Contact + Root Motion 自動處理，減少後期工作
  • ⚡ API 量產 — 5 並發批次自動化，適合建置大規模動作庫
  • 🎮 品質驗證 — Fight Anyone 格鬥遊戲已上線，證明可用於出貨產品
  • 👥 多人追蹤 — 單影片同時提取 5 個角色，適合群戲場景

❌ 缺點

  • 📦 格式限制 — GLB 為主要輸出，FBX 標示 coming soon，BVH 不支援
  • 🎬 固定幀率 — 30fps 無法自訂，部分場景需要 60fps 的開發者需注意
  • 👕 遮擋問題 — 寬鬆衣物、大幅遮擋或極端動作時品質會下降
  • ☁️ 雲端依賴 — 無本地/離線方案，需要網路連線且資料上傳雲端
  • 📹 輸入限制 — 僅接受 MP4 格式，需清晰可見的人物動作
  • 🔄 格式轉換 — 若需 BVH 格式需額外 Blender 轉檔步驟

與 Kimodo 整合方案

Kimodo 用文字描述生成動作（text-to-motion），Viggle 用影片參考提取動作（video-to-mocap）。兩者輸入方式互補，可組合成靈活的混合工作流，依場景選擇最適合的生成方式。

方案 A：影片參考

有參考影片時使用

  • 1. 拍攝或蒐集動作參考影片
  • 2. 上傳至 Viggle API
  • 3. 下載 GLB 骨架動畫
  • 4. Blender retarget 到遊戲角色
  • 5. 匯出 FBX 進入引擎

方案 B：混合工作流

依動作複雜度選擇

  • 簡單/通用動作 → Kimodo text prompt 生成 BVH
  • 複雜/特定動作 → 拍影片 Viggle 提取 GLB
  • 兩者在 Blender 統一 retarget
  • 統一 clean-up（腳步/Root Motion）
  • 匯出到遊戲引擎

方案 C：批次量產

建置大規模動作庫

  • 1. 蒐集大量動作參考影片來源
  • 2. Viggle API 批次處理（$0.03/個）
  • 3. 自動化腳本統一 retarget
  • 4. 品質篩選 + 分類標籤
  • 5. 動作庫建置完成

工作流定位

在整個 AI 動作量產工作流中，Viggle PINOC 佔據「影片參考輸入」這個獨特位置。它與 Kimodo（文字輸入）形成互補，共同構成動作生成的前端，後續再接入統一的 retarget 和 clean-up pipeline。

[動作輸入源]
  ├── Text Prompt ──→ Kimodo（離線生成 BVH）
  ├── 影片參考 ──→ Viggle PINOC（雲端 AI GLB）⭐
  └── 手動 K 幀 ──→ 傳統 DCC（Maya/Blender）

[中間處理 Pipeline]
  BVH Retargeter / ARP Remap → Foot Fix → Root Motion Transfer

[輸出目標]
  Unity / Unreal Engine 5 / Godot → 遊戲產品
