XR Animator

SystemAnimatorOnline — 瀏覽器即時全身動捕

基於 MediaPipe 的開源即時動作捕捉工具，只需瀏覽器與 Webcam 即可進行全身動作追蹤。零安裝、零成本，是最低門檻的動捕入門方案，適合快速預覽動作概念與姿態參考。

MediaPipe免費開源瀏覽器即用VMC ProtocolVRM 模型

核心功能

即時全身追蹤

透過 MediaPipe Pose 與 Holistic 模型，利用單一 Webcam 即時追蹤全身骨架。無需任何穿戴設備或標記點，只要有攝影機的裝置都能使用，大幅降低動作捕捉的技術門檻。

VMC Protocol 串流輸出

支援 VMC Protocol（Virtual Motion Capture Protocol），可將追蹤到的動作資料即時串流至 Unity、Blender 或其他相容軟體。搭配 VRM 格式模型，能即時預覽角色動作效果。

影片展示 XR Animator 如何透過瀏覽器 Webcam 即時捕捉全身動作，並將動作套用到 VRM 3D 角色上。可觀察其追蹤的流暢度與反應延遲狀況。

平台與整合

  • 瀏覽器 — Chrome / Edge / Firefox 等現代瀏覽器即可運行
  • VMC Protocol — 可串流動作資料至 Unity、Blender（需對應接收插件）
  • VRM 模型 — 原生支援載入 VRM 格式角色模型進行即時預覽
  • 作業系統 — Windows / macOS / Linux 皆可，僅需瀏覽器與攝影機

定價方案

完全免費

XR Animator（SystemAnimatorOnline）為開源專案，所有功能完全免費使用。原始碼公開於 GitHub，社群維護。無需註冊帳號、無使用次數限制、無浮水印，是零成本動捕的最佳選擇。

優缺點分析

✓ 優點

  • 零成本零安裝 — 開源免費，瀏覽器直接使用，無任何進入門檻
  • 即時預覽 — 動作追蹤結果立即呈現在 3D 角色上，所見即所得
  • VMC 輸出 — 可串流至專業軟體，延伸應用彈性大
  • 跨平台 — 不限作業系統，任何有瀏覽器的裝置都能用

✗ 缺點

  • 精度有限 — MediaPipe 定位精度不及專業動捕設備，動作容易抖動
  • 無手指追蹤 — 全身模式下不包含手指細節動作捕捉
  • 無腘部追蹤 — 腳部末端追蹤精度不足，步態還原效果差
  • 不適合正式產出 — 品質僅適合概念驗證，無法直接用於遊戲動畫

與 Kimodo 整合

XR Animator 在 Kimodo 動作生產流程中定位為「快速預覽與參考工具」。由於其精度有限且缺乏手指追蹤，不適合作為正式動作產出的來源。但在前期企劃階段，可用來快速測試動作概念是否可行，或作為 Prompt 描述的參考依據。

  • 適用場景 — 動作概念驗證、姿態參考截圖、VMC 即時預覽
  • 不適用 — 正式遊戲動畫產出、需要高精度的動作資料
  • 建議用法 — 先用 XR Animator 確認動作方向，再用 Kimodo 生成正式動作

工作流程定位

在動作生產 Pipeline 中，XR Animator 位於最前端的「概念驗證」階段。它的價值在於讓企劃或動畫師能在幾秒內確認某個動作想法是否值得進一步開發，避免在高成本工具上浪費時間探索不可行的方向。

  • 階段一 — XR Animator 快速驗證動作概念（本工具）
  • 階段二 — 確認方向後，用 Kimodo 生成高品質動作
  • 階段三 — 匯入引擎進行微調與整合

總結與下一步

XR Animator 是零門檻的瀏覽器動捕工具，雖然品質不適合直接產出，但在動作企劃初期有不可替代的快速驗證價值。結合 Kimodo 流程，可建立「快速試 → 正式產」的高效迭代循環。

  • 立即體驗 — 開啟瀏覽器造訪 XR Animator 官網即可使用
  • 設定 VMC — 安裝 Unity/Blender 的 VMC 接收插件，測試串流
  • 整合評估 — 測試其輸出作為 Kimodo prompt 參考的可行性
