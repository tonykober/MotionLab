Expy-Kit — 深度調研報告

工具概述

Expy-Kit 是一款 Blender 外掛，專門用於在不同類型的骨架系統之間進行轉換與動畫 retarget。它作為 Rigify、Mixamo、Unreal Engine、DAZ3D 等不同系統之間的「橋樑」，能夠轉換骨骼命名、建立 Rigify 控制器、批次 retarget 多個動畫。

開發者： Paulo Acampora (pKrime / BallsAndNinjas)

解決的問題：

  • 在 Mixamo → Rigify → 遊戲引擎 的工作流中，快速轉換骨架類型
  • 批次 retarget 大量 Mixamo 動畫到 Rigify 角色
  • 讓 Rigify 角色能直接匯出到遊戲引擎（骨骼命名轉換）
  • 在任意骨架上建立 Rigify 控制器

功能詳細說明

核心功能

功能說明
Convert Bone Names在不同命名系統間轉換骨骼名稱（Rigify ↔ Mixamo ↔ Unreal 等）
Build Rigify Controls在任何骨架上建立 Rigify 控制器（含 IK/FK 切換）
Batch Retarget Actions一次 retarget 數十甚至數百個動畫動作
Make Game Friendly將 Rigify 角色轉換為遊戲引擎相容的簡單骨架
Export to Game Engines匯出至 Unreal Engine / Unity 相容格式
Upload to Mixamo準備角色上傳至 Mixamo 取得動畫
Intermediate FK Rig建立中間 FK 骨架用於 retarget 橋接
DAZ3D 整合支援 DAZ 角色的骨架轉換

支援的骨架系統

  • Rigify
  • Mixamo
  • Unreal Engine Mannequin
  • DAZ3D
  • 自訂骨架（透過 bone_mapping.py 定義）

使用流程（Mixamo → Rigify retarget）

  1. 匯入 Mixamo FBX 動畫
  2. 選取 Rigify 角色
  3. Expy-Kit → Batch Retarget → 選擇所有 Mixamo Actions
  4. 一鍵完成所有動畫的 retarget

使用流程（Rigify → 遊戲引擎匯出）

  1. 選取 Rigify 角色 → Pose Mode
  2. 右鍵 → Make Game Friendly
  3. Convert Bone Names（source: Rigify, target: Mixamo/Unreal）
  4. 匯出 FBX

優缺點分析

✅ 優點

  1. 免費開源 — 核心功能完全免費
  2. 批次 Retarget — 一次處理數百個動畫，非常適合量產工作流
  3. Rigify 深度整合 — 能在任何骨架上建立 Rigify 控制器
  4. 雙向轉換 — Rigify ↔ Mixamo ↔ Unreal 骨骼命名互轉
  5. 遊戲引擎匯出 — Make Game Friendly 一鍵轉換
  6. 快速流線 — 比 ARP/Rokoko 更精簡的 retarget 操作
  7. 社群活躍 — 421 Stars，持續更新
  8. 不需額外付費工具 — 不依賴 Auto-Rig Pro

❌ 缺點

  1. 無 IK Retarget — 純 FK constraint-based retarget，會有腳步滑動
  2. 不直接支援 BVH — 主要設計為 FBX/Mixamo 動畫，BVH 需先匯入再處理
  3. 文件較少 — 主要靠影片教學，無完整文字文件站
  4. Preset 以 Mixamo 為主 — 自訂骨架需修改 bone_mapping.py
  5. 無 Mocap Clean-up 功能 — 純 retarget/轉換工具，不處理動畫品質
  6. Rigify 依賴 — 最佳使用場景需要 Rigify 生態

Pipeline 整合分析

項目內容
輸入來源Blender 場景中的多個 Armature（不同骨架類型：Rigify / ARP / Mixamo / 自訂）
輸出格式Blender Armature Animation（轉移後的動畫 Action）
上游工具Mixamo FBX、ARP 骨架、Rigify 骨架、任何 Blender 內骨架
下游工具ARP Game Export → 引擎；或 Blender 原生 FBX 匯出
推薦搭配Mixamo 動畫 → Expy Kit 轉移到 Rigify/ARP 骨架 → ARP Export → 引擎。跨骨架類型動畫轉移專家。

核心價值：

  • 批次處理能力：AI 量產大量動作時，一次 retarget 數百個
  • Rigify 生態加持：建立控制器後動畫師可用 IK/FK 精修
  • 骨骼命名轉換：最後一哩路的引擎適配

限制：

  • 不解決腳步問題（需搭配 Foot Locker）
  • 不直接吃 BVH（需中間步驟）
  • 更適合 Rigify 工作流，不適合純 Mixamo/ARP 流程
