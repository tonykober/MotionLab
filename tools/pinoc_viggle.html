一、工具概述

PINOC 是 Viggle.ai 推出的 3D Studio 功能模組，專門將普通影片轉換為專業級 Motion Capture 資料。Viggle 原本以 AI 影片迷因生成聞名（「跳舞迷因 App」），2026 年正式轉型推出面向遊戲開發者和 3D 創作者的專業工具。

核心能力：單鏡頭影片 → 65 骨骼 Mixamo 相容骨架 → FBX/GLB 輸出 → 直入遊戲引擎

解決的問題：

  • 傳統 Mocap 系統昂貴（$500-$1,500/clip、$5,000-$15,000/session）
  • 獨立開發者無法負擔動捕設備
  • 手工 K 動作一個走路循環要花一整天
  • 遊戲角色需要大量獨特動作

四、使用流程

基本流程（免費試用）

  1. 前往 3D Studio：https://viggle.ai/3d-studio/landing
  2. Guest 模式可用：無需註冊帳號即可試用
  3. 上傳影片：MP4 格式，拍攝任何人執行動作
  4. AI 處理：自動骨架偵測 + 動作追蹤
  5. 3D 預覽：即時在瀏覽器中查看骨架動畫
  6. 下載：匯出 GLB 檔案
  7. 匯入引擎：拖入 Unity/Unreal/Blender 直接使用

API 流程（量產化）

  1. 取得 API Key：https://portal.viggle.ai/api-keys
  2. 預處理角色（1 credit）
  3. 提交 render job（1 credit/秒）
  4. 下載結果 GLB
  5. 批次整合至 Pipeline

所需素材

  • 手機影片即可（不需專業攝影機）
  • 人物動作清晰可見
  • 支援複雜動作和高速動作

六、優缺點分析

✅ 優點

  1. 極低成本：每個動畫約 $0.03，比傳統 Mocap 便宜 10,000 倍
  2. 免費試用門檻極低：Guest 模式無需帳號，每日 5 次免費
  3. Mixamo 相容骨架：65 骨骼 + Mixamo 命名 → 直入各大引擎
  4. 內建腳部修正：自動 foot contact correction
  5. 有 API + 批次處理：適合量產化
  6. 已有產品驗證：Fight Anyone 遊戲證明品質可用於出貨產品
  7. Root Motion 包含：遊戲角色必備
  8. 多人追蹤：單影片可提取 5 個角色
  9. 引擎相容性完整：Unity/UE5/Godot/Blender/Maya 全支援
  10. 手機拍攝即可：不需要專業設備

❌ 缺點

  1. GLB 為主：FBX 匯出標示「coming soon」，目前主力是 GLB
  2. 不直接輸出 BVH：需要額外格式轉換
  3. 30fps 固定：無法自訂幀率
  4. 複雜動作/寬鬆衣物品質不穩：還有改進空間
  5. 雲端依賴：無離線方案
  6. Video 輸入限制：只接受 MP4
  7. Pre-viz 輸出是影片：只有 3D Studio/Sandbox 才輸出骨架

七、與 Kimodo BVH 工作流的整合可能性

面向評估
格式Kimodo 輸出 BVH，Viggle 輸出 GLB — 都可匯入 Blender 做 retarget。GLB→BVH 轉換容易
互補場景Kimodo = text prompt 生成；Viggle = 影片參考轉 mocap。兩者輸入方式不同，覆蓋不同創作場景
量產化Viggle API 支援批次 + $0.03/animation → 大量參考影片可批次轉換
品質比較需要實測比較：Kimodo vs Viggle 在相同動作的品質差異
Pipeline 整合可以設計：拍攝參考影片 → Viggle API 批次處理 → GLB → Blender retarget → 遊戲引擎
Root MotionViggle 內建 Root Motion，Kimodo 需要額外處理（--no-postprocess 的限制）
腳部修正Viggle 內建 foot contact correction vs Kimodo 目前無後處理

推薦整合方式


方案 A：影片參考工作流
  拍攝動作參考影片 → Viggle API → GLB → Blender retarget → UE5/Unity

方案 B：混合工作流
  簡單動作 → Kimodo text prompt → BVH
  複雜/特定動作 → 拍攝影片 → Viggle → GLB
  兩者統一在 Blender 做 retarget + clean-up → 遊戲引擎

方案 C：量產工作流
  蒐集大量動作參考影片（YouTube/自拍/素材庫）
  → Viggle API 批次處理（$0.03/個）
  → 統一 retarget 到遊戲角色骨架
  → 動作庫建置
